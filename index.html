# Mock door (`mockdoor.houseofloulous.com`) The mock app is a **single static file**: **`public/mock-door.html`** in this repo (same HTML you can publish anywhere). **Only difference from live:** **fake guest names** and **stock face photos** (placeholder strangers from [pravatar.cc](https://pravatar.cc), not your guests). **NDA Google Form** is the same as production and works fine for practice. No Airtable / no live list. This app does not open `door.houseofloulous.com`. Dummy `tel:` / `mailto:` use 555 / `@example.com`. --- ## Easiest setup (recommended): small GitHub repo + GitHub Pages This matches how **`loulous-doorlist`** hosts the real door: plain HTML, no Vercel/Next required for the mock. ### 1. Create a new repository on GitHub - Name it something obvious, e.g. **`loulous-mock-door`** (or `houseofloulous-mock-door`). - Can be **public** or **private** ‚Äî your choice. - **Do not** reuse `loulous-doorlist` as the site root for two different domains (one repo = one custom domain for Pages is simplest). A **second repo** keeps the mock app clearly separate from the live door repo. ### 2. Add the mock as `index.html` + icon - In **`loulous-portal`**, open **`public/mock-door.html`**. - Copy **the entire file**. - In the **new** repo, create **`index.html`** and paste (same content; GitHub Pages serves `index.html` at the site root). - Also upload **`public/mock-app-icon.png`** to the **same repo root** (rename not needed). The mock app uses it for the **tab icon** and **Add to Home Screen** so it‚Äôs visually distinct from the real door. If you skip this file, browsers may show a generic icon. ### 3. Add a `CNAME` file (for `mockdoor.houseofloulous.com`) In the **root** of that new repo, add a file named **`CNAME`** (no extension) with **one line**: ```text mockdoor.houseofloulous.com ``` Commit and push to **`main`**. ### 4. Turn on GitHub Pages - Repo ‚Üí **Settings** ‚Üí **Pages**. - **Source:** Deploy from branch **`main`**, folder **`/ (root)`**. - Save. After a minute or two, GitHub shows a URL (often `https://houseofloulous.github.io/loulous-mock-door/` until the custom domain works). ### 5. DNS (where `houseofloulous.com` is managed) Add a **CNAME** record: | Type | Host / Name | Target | |-------|-------------|--------| | CNAME | `mockdoor` | `houseofloulous.github.io` | (If your DNS provider wants a trailing dot, use `houseofloulous.github.io.` ‚Äî they‚Äôll say so in the UI.) **Do not** point `mockdoor` at the same place as `door` unless you know what you‚Äôre doing ‚Äî for this setup it should point at **GitHub‚Äôs** hostname above, not at your door host. ### 6. Custom domain in GitHub - **Settings** ‚Üí **Pages** ‚Üí **Custom domain:** enter `mockdoor.houseofloulous.com`, save. - Wait for DNS + **Enforce HTTPS** when GitHub offers it. After DNS validates, GitHub may show **‚ÄúRequesting a certificate‚Ä¶‚Äù** ‚Äî that‚Äôs normal; issuance can take **a few minutes up to about an hour**, then **Enforce HTTPS** becomes available. ### If GitHub says ‚ÄúDNS check unsuccessful‚Äù / `InvalidDNSError` 1. **Confirm the CNAME at your DNS host** (where `houseofloulous.com` DNS is edited): - **Name / host:** `mockdoor` (sometimes shown as `mockdoor.houseofloulous.com`). - **Type:** `CNAME`. - **Target / value:** `houseofloulous.github.io` (add a trailing `.` only if the UI asks for FQDN form). 2. **Remove conflicts:** There must be **no** `A` or `AAAA` record for `mockdoor` at the same time as the CNAME (only the CNAME for this subdomain). 3. **Only one host for `mockdoor`:** If you also added `mockdoor.houseofloulous.com` in **Vercel** (or another provider), pick **either** GitHub Pages **or** Vercel and point DNS to **one** target ‚Äî not both. 4. **Propagation:** After saving DNS, wait a few minutes to 48 hours; use **Check again** in GitHub, or run `dig mockdoor.houseofloulous.com CNAME` / an online ‚ÄúDNS checker‚Äù to see if the CNAME is visible publicly. 5. **Cloudflare:** If the domain is on Cloudflare, try **DNS only** (grey cloud) for the `mockdoor` record while GitHub verifies; you can tune later. ### If `mockdoor` loads the **real** door (wrong app) Those issues are separate from the yellow ‚ÄúDNS check‚Äù banner ‚Äî your site can still resolve while GitHub‚Äôs checker is picky. 1. **Follow redirects (see where you actually land)** In Terminal: ```bash curl -sIL http://mockdoor.houseofloulous.com | head -40 curl -sIL https://mockdoor.houseofloulous.com | head -40 ``` - If you see **`Location:`** pointing to **`door.`** (or another host), something **in front of GitHub** is redirecting: check **Cloudflare Page Rules / Redirect Rules**, **registrar ‚ÄúURL forwarding‚Äù**, or remove **`mockdoor`** from **Vercel** (and any other host) if you‚Äôre standardizing on **GitHub Pages** only. 2. **Pick one host for `mockdoor` (GitHub *or* Vercel)** - **GitHub Pages:** DNS CNAME `mockdoor` ‚Üí `houseofloulous.github.io` ‚Äî then **remove** `mockdoor.houseofloulous.com` from the Vercel project‚Äôs **Domains** so nothing there hijacks or rewrites traffic. - **Vercel:** DNS CNAME `mockdoor` ‚Üí `cname.vercel-dns.com` (or whatever Vercel shows) ‚Äî then **don‚Äôt** expect the mock repo on GitHub Pages to own that hostname. 3. **URL stays `mockdoor` but content looks like production** The **`loulous-mock-door`** repo‚Äôs **`index.html`** may be the wrong paste (e.g. real `index.html` from `loulous-doorlist`). Replace it with the full contents of **`public/mock-door.html`** from this repo; the browser tab title is **‚ÄúMock Door‚Äù**, the select screen shows **MOCK APP** under the logo, and the mock password is **`practice`**. ### 7. Test - Open **`https://mockdoor.houseofloulous.com/`** ‚Äî you should see **MOCK APP** on the select screen, password **`practice`**, fake guests only. - Real door stays on **`https://door.houseofloulous.com/`** (or whatever `loulous-doorlist` uses) ‚Äî different URL, different repo. ### Updating the mock later When you change **`public/mock-door.html`** here, copy it again to the mock repo‚Äôs **`index.html`**, commit, and push. --- ## Option B: Members portal repo + Vercel (already using `loulous-portal`) If this Next app is already deployed to Vercel: - File stays **`public/mock-door.html`**. - **`src/proxy.ts`** rewrites the **root URL** (`/`) on **`mockdoor.houseofloulous.com`** (and **`mockdoor.localhost`**) to **`/mock-door.html`**. - In Vercel: add domain **`mockdoor.houseofloulous.com`** to **this** project. - DNS: CNAME **`mockdoor`** ‚Üí the hostname Vercel gives you (e.g. `cname.vercel-dns.com`), **not** the same record as `door` unless both are configured in Vercel. Use **either** GitHub Pages (A) **or** Vercel (B) for `mockdoor` ‚Äî not two different hosts fighting over the same subdomain. --- ## Quick comparison | | Real door | Mock app | |---|-----------|----------| | **Typical URL** | `door.houseofloulous.com` | `mockdoor.houseofloulous.com` | | **Repo** | `loulous-doorlist` | e.g. `loulous-mock-door` (Pages) **or** this repo (Vercel) | | **Data** | Airtable | None ‚Äî fake names in the HTML | | **Password** | Real door password | `practice` (mock app only) | --- ## Local testing (this Next repo) ```bash # optional: map a fake host # /etc/hosts ‚Üí 127.0.0.1 mockdoor.localhost npm run dev # open http://mockdoor.localhost:3000/ ``` `src/proxy.ts` rewrites **`/`** on `mockdoor.localhost` the same way as `mockdoor.houseofloulous.com`.
